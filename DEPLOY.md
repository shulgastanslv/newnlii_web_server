# Инструкция по деплою сервера и настройке внешнего доступа

## Быстрый старт

### 1. Установка зависимостей

```bash
# Создание виртуального окружения
python -m venv .venv

# Активация (Windows)
.venv\Scripts\activate

# Активация (Linux/Mac)
source .venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/dbname
CORS_ORIGINS=http://localhost:3000,https://yourdomain.com
```

**Важно:** Для внешнего доступа установите `CORS_ORIGINS=*` (разрешает все домены) или укажите конкретные домены через запятую.

### 3. Запуск сервера для внешнего доступа

#### Windows:
```bash
run_production.bat
```

#### Linux/Mac:
```bash
chmod +x run_production.sh
./run_production.sh
```

#### Или вручную:
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

**Ключевые параметры:**
- `--host 0.0.0.0` - сервер будет слушать на всех сетевых интерфейсах (не только localhost)
- `--port 8000` - порт сервера
- `--workers 4` - количество worker процессов (опционально, для production)

## Настройка доступа извне

### Вариант 1: Прямой доступ через IP (локальная сеть)

1. **Узнайте ваш локальный IP адрес:**
   - Windows: `ipconfig` (смотрите IPv4 Address)
   - Linux/Mac: `ifconfig` или `ip addr`

2. **Запустите сервер** с `--host 0.0.0.0`

3. **Откройте порт в брандмауэре Windows:**
   ```powershell
   # Запустите PowerShell от имени администратора
   New-NetFirewallRule -DisplayName "Devsy API Server" -Direction Inbound -LocalPort 8000 -Protocol TCP -Action Allow
   ```

4. **Доступ к серверу:**
   - Из локальной сети: `http://YOUR_LOCAL_IP:8000`
   - Пример: `http://192.168.1.100:8000`

### Вариант 2: Использование ngrok (для тестирования)

1. **Установите ngrok:**
   - Скачайте с https://ngrok.com/
   - Или через chocolatey: `choco install ngrok`

2. **Запустите сервер:**
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8000
   ```

3. **В другом терминале запустите ngrok:**
   ```bash
   ngrok http 8000
   ```

4. **Используйте URL от ngrok** (например: `https://abc123.ngrok.io`)

### Вариант 3: Настройка роутера (для постоянного доступа)

1. **Настройте статический IP** для вашего компьютера в роутере

2. **Настройте Port Forwarding:**
   - Войдите в настройки роутера (обычно 192.168.1.1)
   - Найдите раздел "Port Forwarding" или "Виртуальные серверы"
   - Добавьте правило:
     - Внешний порт: 8000
     - Внутренний IP: IP вашего компьютера
     - Внутренний порт: 8000
     - Протокол: TCP

3. **Узнайте ваш внешний IP:**
   - Откройте https://whatismyipaddress.com/
   - Или выполните: `curl ifconfig.me`

4. **Доступ к серверу:**
   - `http://YOUR_EXTERNAL_IP:8000`

**⚠️ ВНИМАНИЕ:** Открытие портов в роутере делает ваш сервер доступным из интернета. Убедитесь, что:
- У вас настроена аутентификация
- Используется HTTPS (рекомендуется)
- Сервер защищен от атак

### Вариант 4: Использование облачного сервиса

Для production рекомендуется использовать:
- **Heroku** - простой деплой
- **DigitalOcean** - VPS серверы
- **AWS EC2** - масштабируемая инфраструктура
- **Railway** - простой деплой с автоматическим HTTPS
- **Render** - бесплатный хостинг с автоматическим деплоем

## Настройка CORS для внешнего доступа

В файле `.env` установите:

```env
# Разрешить все домены (для разработки)
CORS_ORIGINS=*

# Или указать конкретные домены
CORS_ORIGINS=https://yourdomain.com,https://app.yourdomain.com
```

## Проверка работы сервера

1. **Локально:**
   ```bash
   curl http://localhost:8000
   ```

2. **Извне (замените на ваш IP/домен):**
   ```bash
   curl http://YOUR_IP:8000
   ```

3. **Документация API:**
   - Swagger UI: `http://YOUR_IP:8000/docs`
   - ReDoc: `http://YOUR_IP:8000/redoc`

## Безопасность

### Рекомендации для production:

1. **Используйте HTTPS:**
   - Настройте reverse proxy (nginx) с SSL сертификатом
   - Или используйте облачный сервис с автоматическим HTTPS

2. **Настройте аутентификацию:**
   - Добавьте JWT токены
   - Используйте API ключи

3. **Ограничьте CORS:**
   - Не используйте `CORS_ORIGINS=*` в production
   - Укажите только разрешенные домены

4. **Настройте rate limiting:**
   - Ограничьте количество запросов с одного IP

5. **Используйте переменные окружения:**
   - Не храните секреты в коде
   - Используйте `.env` файл (не коммитьте его в git!)

## Troubleshooting

### Проблема: Сервер не доступен извне

1. Проверьте, что сервер запущен с `--host 0.0.0.0`
2. Проверьте настройки брандмауэра
3. Проверьте настройки роутера (port forwarding)
4. Убедитесь, что порт не занят другим приложением

### Проблема: CORS ошибки

1. Проверьте настройку `CORS_ORIGINS` в `.env`
2. Убедитесь, что клиент отправляет правильные заголовки
3. Проверьте, что домен клиента указан в `CORS_ORIGINS`

### Проблема: Подключение к базе данных

1. Проверьте `DATABASE_URL` в `.env`
2. Убедитесь, что база данных запущена
3. Проверьте права доступа к базе данных
