# 🔍 Как узнать адрес для доступа к серверу

## Быстрая проверка

Запустите скрипт:
```bash
check_server_address.bat
```

Он покажет все доступные адреса.

## Ручная проверка

### 1. Локальный доступ (с этого же компьютера)

**Адрес:** `http://localhost:8000` или `http://127.0.0.1:8000`

Проверка:
```bash
curl http://localhost:8000
```

### 2. Доступ из локальной сети (другие устройства в той же сети Wi-Fi)

**Шаг 1: Узнайте ваш локальный IP адрес**

**Windows:**
```bash
ipconfig
```
Ищите строку **"IPv4 Address"** в разделе вашего сетевого адаптера (Ethernet или Wi-Fi).

Пример вывода:
```
IPv4 Address. . . . . . . . . . . . : 192.168.1.100
```

**Шаг 2: Используйте этот IP**
- Адрес: `http://192.168.1.100:8000` (замените на ваш IP)

**Шаг 3: Откройте порт в брандмауэре**
```powershell
# Запустите PowerShell от имени администратора
New-NetFirewallRule -DisplayName "Devsy API" -Direction Inbound -LocalPort 8000 -Protocol TCP -Action Allow
```

**Шаг 4: Проверка с другого устройства**
- Откройте браузер на телефоне/планшете в той же Wi-Fi сети
- Перейдите по адресу: `http://ВАШ_IP:8000`
- Пример: `http://192.168.1.100:8000`

### 3. Доступ из интернета

#### Вариант A: Через ngrok (самый простой)

1. **Установите ngrok:**
   - Скачайте: https://ngrok.com/download
   - Или: `choco install ngrok`

2. **Запустите сервер:**
   ```bash
   run_production.bat
   ```

3. **В другом терминале запустите ngrok:**
   ```bash
   ngrok http 8000
   ```

4. **Скопируйте адрес из ngrok:**
   ```
   Forwarding  https://abc123.ngrok.io -> http://localhost:8000
   ```
   
   **Используйте:** `https://abc123.ngrok.io` (ваш адрес будет другой)

#### Вариант B: Через внешний IP (требует настройки роутера)

1. **Узнайте ваш внешний IP:**
   ```bash
   # Через PowerShell
   (Invoke-WebRequest -Uri 'https://api.ipify.org' -UseBasicParsing).Content
   
   # Или откройте в браузере
   https://whatismyipaddress.com/
   ```

2. **Настройте Port Forwarding в роутере:**
   - Войдите в роутер (обычно `192.168.1.1` или `192.168.0.1`)
   - Найдите "Port Forwarding" / "Виртуальные серверы"
   - Добавьте правило:
     - **Внешний порт:** 8000
     - **Внутренний IP:** ваш локальный IP (из пункта 2)
     - **Внутренний порт:** 8000
     - **Протокол:** TCP

3. **Используйте внешний IP:**``
   - Адрес: `http://ВАШ_ВНЕШНИЙ_IP:8000`
   - Пример: `http://95.123.45.67:8000`

⚠️ **ВНИМАНИЕ:** Открытие портов делает сервер доступным из интернета. Убедитесь в безопасности!

## Проверка работы сервера

### Проверка, что сервер запущен:

```bash
# Проверка порта
netstat -an | findstr :8000
```

Если видите `0.0.0.0:8000` или `127.0.0.1:8000` - сервер работает.

### Тестирование доступа:

**Локально:**
```bash
curl http://localhost:8000
```

**Из браузера:**
- Откройте: `http://localhost:8000/docs` (Swagger документация)
- Или: `http://localhost:8000/redoc` (ReDoc документация)

**С другого устройства:**
- Откройте браузер
- Введите: `http://ВАШ_IP:8000/docs`

## Частые проблемы

### ❌ "Сервер недоступен"

**Решение:**
1. Убедитесь, что сервер запущен с `--host 0.0.0.0`
2. Проверьте брандмауэр Windows
3. Проверьте, что порт не занят: `netstat -an | findstr :8000`

### ❌ "Connection refused" извне

**Решение:**
1. Проверьте Port Forwarding в роутере
2. Убедитесь, что используете правильный IP
3. Проверьте настройки брандмауэра

### ❌ CORS ошибки

**Решение:**
1. В файле `.env` установите: `CORS_ORIGINS=*`
2. Перезапустите сервер

## Полезные команды

```bash
# Узнать все IP адреса
ipconfig

# Узнать внешний IP
curl https://api.ipify.org

# Проверить, слушается ли порт
netstat -an | findstr :8000

# Проверить доступность сервера
curl http://localhost:8000
```
