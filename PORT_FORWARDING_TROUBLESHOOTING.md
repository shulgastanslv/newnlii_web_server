# Решение проблем с пробросом портов

## Ваша ситуация
- Сервер запущен на `192.168.0.106:8000`
- Настроен проброс портов на роутере
- Порт 8000 открыт в брандмауэре Windows
- Но внешний IP не переадресовывает на локальный

## Основные причины и решения

### 1. **Hairpin NAT (NAT Loopback) - САМАЯ ЧАСТАЯ ПРИЧИНА**

**Проблема:** Многие роутеры не поддерживают обращение к внешнему IP с того же компьютера, который находится внутри сети.

**Как проверить:**
- Попробуйте обратиться к внешнему IP с другого устройства (телефон через мобильный интернет, другой компьютер в другой сети)
- Если с внешнего устройства работает, а с вашего компьютера нет - это Hairpin NAT

**Решения:**
- ✅ Используйте локальный IP `192.168.0.106:8000` для доступа из локальной сети
- ✅ Используйте внешний IP только для доступа из интернета
- ✅ Некоторые роутеры поддерживают Hairpin NAT - включите эту опцию в настройках

### 2. **Неправильная настройка роутера**

**Проверьте в настройках роутера:**

```
Внутренний IP (Internal IP / Local IP): 192.168.0.106
Внутренний порт (Internal Port / Local Port): 8000
Внешний порт (External Port / Public Port): 8000
Протокол: TCP (или TCP/UDP)
Статус: Включено (Enabled)
```

**Важно:**
- IP должен быть статическим (зарезервированным в DHCP или настроенным вручную)
- Убедитесь, что правило активно и сохранено
- Перезагрузите роутер после настройки

### 3. **CGNAT (Carrier-Grade NAT)**

**Проблема:** Ваш интернет-провайдер может использовать CGNAT, что блокирует входящие соединения из интернета.

**Как проверить:**
- Зайдите в настройки роутера и посмотрите WAN IP
- Сравните с вашим внешним IP (проверьте на https://api.ipify.org)
- Если они разные - у вас CGNAT

**Решения:**
- Обратитесь к провайдеру с запросом на выделение "белого" IP (обычно платная услуга)
- Используйте VPN с выделенным IP
- Используйте ngrok или другие туннелирующие сервисы

### 4. **Брандмауэр Windows**

**Проверьте правила:**

1. Откройте "Брандмауэр Защитника Windows"
2. Дополнительные параметры
3. Правила для входящих подключений
4. Найдите правило для порта 8000

**Если правила нет, создайте:**
```cmd
netsh advfirewall firewall add rule name="Devsy Server Port 8000" dir=in action=allow protocol=TCP localport=8000
```

**Или через GUI:**
- Создать правило → Для порта → TCP → 8000 → Разрешить подключение

### 5. **Брандмауэр роутера**

**Проверьте:**
- Настройки брандмауэра на роутере
- Убедитесь, что порт 8000 разрешен для входящих соединений
- Некоторые роутеры требуют отдельного разрешения в брандмауэре

### 6. **Сервер не слушает на всех интерфейсах**

**Проверьте:**
```cmd
netstat -an | findstr :8000
```

Должно быть:
```
TCP    0.0.0.0:8000           0.0.0.0:0              LISTENING
```

Если видите `127.0.0.1:8000` вместо `0.0.0.0:8000` - сервер слушает только localhost.

**Решение:** Убедитесь, что запускаете через `run_production.bat`, который использует `--host 0.0.0.0`

### 7. **Динамический IP адрес**

**Проблема:** Внешний IP может меняться.

**Решение:** Используйте DDNS сервис (DynDNS, No-IP, DuckDNS) для получения постоянного доменного имени.

## Пошаговая диагностика

### Шаг 1: Запустите диагностический скрипт
```cmd
diagnose_port_forwarding.bat
```

### Шаг 2: Проверьте локальный доступ
```cmd
curl http://localhost:8000
# или
curl http://192.168.0.106:8000
```

### Шаг 3: Проверьте доступ из локальной сети
С другого устройства в той же сети:
```
http://192.168.0.106:8000
```

### Шаг 4: Проверьте внешний доступ
С устройства вне вашей сети (телефон через мобильный интернет):
```
http://ВАШ_ВНЕШНИЙ_IP:8000
```

### Шаг 5: Проверьте открытость порта онлайн
Используйте сервисы:
- https://www.yougetsignal.com/tools/open-ports/
- https://canyouseeme.org/
- https://www.portchecker.co/

Введите порт 8000 и ваш внешний IP.

## Быстрое решение для тестирования

Если нужно быстро протестировать доступ из интернета, используйте ngrok:

```cmd
ngrok http 8000
```

Это создаст временный туннель и даст вам публичный URL.

## Проверка настроек роутера

### Типичные названия разделов в роутерах:
- Port Forwarding
- Virtual Server
- NAT Forwarding
- Applications & Gaming
- Firewall → Port Forwarding

### Что должно быть настроено:
```
Service Name: Devsy Server (любое имя)
External Port: 8000
Internal Port: 8000
Internal IP: 192.168.0.106
Protocol: TCP
Status: Enabled
```

## Дополнительные проверки

### Проверка, что IP статический:
```cmd
ipconfig /all
```

Найдите ваш сетевой адаптер и проверьте:
- DHCP включен → зарезервируйте IP в роутере
- Или настройте статический IP вручную

### Проверка правил брандмауэра:
```cmd
netsh advfirewall firewall show rule name=all | findstr 8000
```

### Проверка активных соединений:
```cmd
netstat -an | findstr :8000
```

## Если ничего не помогает

1. **Перезагрузите роутер** - часто помогает
2. **Временно отключите брандмауэр Windows** для теста
3. **Проверьте логи роутера** - могут показать, почему соединения блокируются
4. **Обратитесь к провайдеру** - возможно, они блокируют входящие соединения
5. **Используйте альтернативные решения:**
   - ngrok для тестирования
   - VPN сервер
   - Облачный сервер

## Контакты для помощи

Если проблема не решается, соберите следующую информацию:
- Результат `diagnose_port_forwarding.bat`
- Модель роутера
